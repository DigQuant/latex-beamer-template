### 2.2 重点项目执行详情

#### 内蒙古财经大学项目
- 工作投入：备课 32 小时，授课 8 小时（线上 4 天）
- 产出成果：毕业论文选题辅助工具（ThesisTopicAssistant）、AI 生成文本检测工具（AiGeneratedTextDetector）
- 主要问题：首次线上授课时间把控不足；课件内容过长且修改困难；低估学生水平导致案例设计冗余；课件知识密度偏低，不利于自学与新教师上手

#### 珠海科技学院项目
- 工作投入：备课 16 小时，授课 16 小时（线下 2 天）
- 产出成果：构建 1500 条 BOSS 直聘岗位数据集与 5991 条淘宝商品评价数据集；开发岗位技能挖掘实战案例（JobSkillMining）
- 主要问题：学生基础差异显著，部分学员零基础，教学重心被迫转向大模型辅助；线下机房环境导致教师走动频繁，线上助教体验不佳；作业收集流程不完善，缺乏学生身份识别机制

#### 山西大学项目
- 工作投入：备课 88 小时，授课 32 小时（线下 5 天）+ 答辩 3 小时
- 产出成果：
  - 文本生成：特定歌手风格歌词生成模型（LyricLstm, LyricTransformer）
  - 数据分析：大模型岗位技能聚类分析（JobSkillEmbeddingAnalysis）
  - 应用开发：本地文档智能问答系统（LocalDocRag）
  - 智能 Agent：AI 智能阅卷评分系统（AutoGradingAgent）
  - 多模态：音色克隆技术实践（VoiceCloning）
- 主要问题：学生能力跨度大但均有编程基础；大模型相关库接口频繁变更，需引入专业 Python 项目管理工具；学生本地环境问题频发（系统文件缺失、驱动过时、磁盘空间不足、代理配置异常等）

#### 师资培训项目
- 工作投入：备课 32 小时，授课 8 小时（线下 2 天）
- 产出成果：智能数据探索工具（DataFormulator）、Kaggle 数据集与代码笔记语义检索工具（KaggleRag）
- 主要问题：部分参训教师无代码基础，需临时调整教学内容，导致休息时间严重不足；需优化课程顺序以兼顾不同基础学员的学习节奏

#### 韩山师范学院项目
- 工作投入：备课 4 小时，授课 16 小时（线上 2 天）
- 平台问题（Ai-Lab）：
  - 学生名单导入错误导致无法参与小测
  - CCI 时长资源不足需手动续时
  - 代码单元格无法编辑 Markdown 单元格引发系统假死
  - 作业缺乏批量下载与评分功能，需通过抓包手动解析

## 五、反思与改进方向

### 5.1 个人层面反思
- 价值观定位：坚持差异化创新，以兴趣驱动工作，注重务实落地与全球视野融合
- 沟通优化：需警惕“知识的诅咒”，加强换位思考能力，避免因预设听众背景导致教学与沟通障碍

### 5.2 部门层面建议
- 技术资源支持：建议建立常态化内部技术交流机制，提供 API Key 配额或显卡硬件资源支持
- 标准化建设：当前缺乏统一的课程开发规范，导致后期维护成本增加
- 数据安全风险：
  - Ai-Lab 华为对象存储未限制目录索引，存在 80.3 GB 文件可被遍历泄露的风险
  - 点宽杯参赛作品中发现未失效的第三方 API Key，需建立提交内容安全审查机制
- 成本优化：对象存储外网流量费用较高，建议引入 CDN 服务进行流量分发优化
